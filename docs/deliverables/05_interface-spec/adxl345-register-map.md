# ADXL345 レジスタマップ仕様書

| 項目 | 内容 |
|---|---|
| ドキュメント番号 | IF-ADXL-001 |
| バージョン | 1.0 |
| 対象デバイス | Analog Devices ADXL345（3軸 デジタル加速度センサ） |
| 接続 | SPI（4線式, MODE 3） |

> **位置づけ** — MCP3008（[IF-SPI-001](spi-hardware-if.md)）が「レジスタを持たないコマンド型 ADC」だったのに対し、
> 本デバイスは**レジスタマップを持つデジタルセンサ**であり、設定を**ビット単位**で行う。
> 本書のレジスタ表・ビットフィールド図は実装ヘッダ
> [`libadxl345/include/adxl345_reg.hpp`](../../../libadxl345/include/adxl345_reg.hpp) と 1:1 で対応する。

---

## 1. 物理インターフェース

| 信号名 | 方向 | RPi 3B+ ピン | ADXL345 ピン | 説明 |
|---|---|---|---|---|
| SCLK | Master → Slave | GPIO11 (Pin 23) | SCL/SCLK | クロック |
| MOSI | Master → Slave | GPIO10 (Pin 19) | SDA/SDI   | マスタ送信（アドレス＋データ） |
| MISO | Slave → Master | GPIO9  (Pin 21) | SDO       | マスタ受信（レジスタ値） |
| CE0  | Master → Slave | GPIO8  (Pin 24) | CS        | チップセレクト（アクティブ Low） |

> VDD/VS は 3.3V、GND は GND に接続する。

## 2. 電気的仕様

| 項目 | 仕様 |
|---|---|
| 動作電圧 | 3.3V |
| 最大クロック | 5 MHz（本実装は安全側で 1 MHz） |
| データフォーマット | 8ビット、MSBファースト |
| SPIモード | MODE 3（CPOL=1, CPHA=1） |

## 3. SPI アクセスプロトコル

各転送の先頭 1 バイトは **アドレスバイト** で、上位 2bit でアクセス種別を指定する。

```
アドレスバイト
┌──────┬──────┬──────────────────────────────┐
│ bit7 │ bit6 │ bit5      .. アドレス ..   bit0│
│ R/W  │  MB  │        レジスタアドレス         │
└──────┴──────┴──────────────────────────────┘
  R/W : 1=読み出し / 0=書き込み
  MB  : 1=連続(マルチバイト)転送 / 0=単一バイト
```

| 操作 | TX（MOSI） | RX（MISO） |
|---|---|---|
| 単一レジスタ読み出し | `[ 0x80\|addr, 0x00 ]` | `[ _, value ]` |
| 単一レジスタ書き込み | `[ 0x00\|addr, value ]` | `[ _, _ ]` |
| 連続読み出し（n バイト） | `[ 0xC0\|addr, 0x00×n ]` | `[ _, d0, d1, … ]` |

例: データ 6 バイト一括読み出しのアドレスバイトは `0x80 | 0x40 | 0x32 = 0xF2`。

## 4. レジスタマップ（本実装で使用するもの）

| アドレス | 名称 | R/W | リセット値 | 説明 |
|---|---|---|---|---|
| 0x00 | DEVID       | R   | 0xE5 | デバイス ID（固定値。疎通・誤配線検出に使用） |
| 0x2C | BW_RATE     | R/W | 0x0A | 出力データレート / 低電力モード |
| 0x2D | POWER_CTL   | R/W | 0x00 | 電源・測定制御 |
| 0x31 | DATA_FORMAT | R/W | 0x00 | データフォーマット（分解能・レンジ等） |
| 0x32–0x37 | DATAX0..DATAZ1 | R | 0x00 | X/Y/Z 各 16bit（リトルエンディアン, 連続 6 バイト） |

## 5. ビットフィールド定義（ビットマップ）

### 5.1 POWER_CTL (0x2D)

```
 bit:  7      6      5      4          3        2      1      0
     ┌──────┬──────┬──────┬──────────┬────────┬──────┬─────────────┐
     │  0   │  0   │ LINK │AUTO_SLEEP│MEASURE │SLEEP │   WAKEUP    │
     └──────┴──────┴──────┴──────────┴────────┴──────┴─────────────┘
```

| ビット | 名称 | 意味 |
|---|---|---|
| 5 | LINK | Link 機能（アクティビティ/非アクティビティの連動） |
| 4 | AUTO_SLEEP | 自動スリープ |
| 3 | **MEASURE** | 0=待機(standby) / **1=測定(measure)** ← 測定開始に必須 |
| 2 | SLEEP | スリープ |
| 1:0 | WAKEUP | スリープ時のサンプリング周波数（8/4/2/1 Hz） |

### 5.2 DATA_FORMAT (0x31)

```
 bit:  7         6         5         4    3        2        1      0
     ┌─────────┬─────────┬─────────┬───┬────────┬────────┬─────────────┐
     │SELF_TEST│SPI 3wire│INT_INV  │ 0 │FULL_RES│JUSTIFY │   RANGE     │
     └─────────┴─────────┴─────────┴───┴────────┴────────┴─────────────┘
```

| ビット | 名称 | 意味 |
|---|---|---|
| 7 | SELF_TEST | セルフテスト印加 |
| 6 | SPI (3wire) | 1=3線式 / 0=4線式 SPI |
| 5 | INT_INVERT | 割り込み極性反転 |
| 3 | **FULL_RES** | 1=フル分解能（**3.9 mg/LSB 固定**）/ 0=10bit 固定 |
| 2 | JUSTIFY | 1=左詰め(MSB) / 0=右詰め（符号拡張） |
| 1:0 | **RANGE** | 00=±2g / 01=±4g / 10=±8g / **11=±16g** |

### 5.3 BW_RATE (0x2C)

```
 bit:  7   6   5    4          3      2      1      0
     ┌───┬───┬───┬──────────┬─────────────────────────┐
     │ 0 │ 0 │ 0 │LOW_POWER │          RATE           │
     └───┴───┴───┴──────────┴─────────────────────────┘
```

| ビット | 名称 | 意味 |
|---|---|---|
| 4 | LOW_POWER | 低電力モード |
| 3:0 | RATE | 出力データレート（既定 0x0A = 100 Hz） |

## 6. 初期化シーケンス（本実装 `Adxl345::open()`）

```
1. SPI を MODE 3 / 1MHz / 8bit でオープン
2. DEVID(0x00) を読む → 0xE5 でなければ失敗（close）
3. DATA_FORMAT(0x31) ← FULL_RES | RANGE_16G  (= 0x0B)
4. POWER_CTL(0x2D)   ← MEASURE                (= 0x08)   ← 測定開始
```

## 7. 変換式

フル分解能（FULL_RES=1）では、レンジに依らずスケール係数は一定:

```
raw_axis = (int16_t)((DATAn1 << 8) | DATAn0)   // 符号付き 16bit, リトルエンディアン
accel[g] = raw_axis * 3.9e-3                    // 3.9 mg/LSB
```
