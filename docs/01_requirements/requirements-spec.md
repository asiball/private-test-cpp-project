## 1. 目的

Linux組み込みシステム向けのSPIデバイス制御ソフトウェアを開発し、発注者のプロダクトに組み込み可能な形で納品する。

## 2. 開発範囲

| コンポーネント | 説明 |
|---|---|
| SpiDriver | `/dev/spidev` を介したSPI送受信の低レベル制御 |
| libdevice | ユーザ空間向け高レベルAPIを提供する共有ライブラリ |
| device-ctl | 動作確認・デバッグ用CLIツール |

## 3. 機能要件

### 3.1 SpiDriver

| ID | 要件 | 優先度 |
|---|---|---|
| FR-DRV-001 | `/dev/spidev` のオープン・クローズができること | 必須 |
| FR-DRV-002 | SPI MODE 0〜3 を設定できること | 必須 |
| FR-DRV-003 | 転送速度を最大 2MHz まで設定できること（CHG-001） | 必須 |
| FR-DRV-004 | 8ビット・フルデュプレクス転送ができること | 必須 |
| FR-DRV-005 | EAGAIN 発生時に最大3回リトライすること（CHG-002） | 必須 |

### 3.2 libdevice

| ID | 要件 | 優先度 |
|---|---|---|
| FR-LIB-001 | 任意のレジスタアドレスから指定バイト数を同期読み出しできること | 必須 |
| FR-LIB-002 | 任意のレジスタアドレスへバイト列を同期書き込みできること | 必須 |
| FR-LIB-003 | 非同期読み出しAPIを提供すること（CHG-003） | 必須 |
| FR-LIB-004 | 動的共有ライブラリ（.so）として提供すること | 必須 |

### 3.3 device-ctl

| ID | 要件 | 優先度 |
|---|---|---|
| FR-CLI-001 | `read <reg> <len>` コマンドで同期読み出しができること | 必須 |
| FR-CLI-002 | `write <reg> <byte>...` コマンドで書き込みができること | 必須 |
| FR-CLI-003 | `-d <device>` でSPIデバイスパスを指定できること | 必須 |
| FR-CLI-004 | `--async` フラグで非同期readを実行できること（CHG-004） | 必須 |

## 4. 非機能要件

| ID | 要件 |
|---|---|
| NFR-001 | C++11 準拠（GCC 7.5でコンパイル可能であること） |
| NFR-002 | Linux kernel 5.10 / Raspberry Pi 4B で動作すること |
| NFR-003 | 単体テストのカバレッジ 80% 以上 |
| NFR-004 | 共有ライブラリはメジャーバージョン固定でABIを維持すること |

## 5. 制約事項

- 本ソフトウェアはLinux専用とする（Windows/macOS対応は対象外）
- カーネルモジュールは対象外（ユーザ空間のみ）
- サードパーティライブラリは標準ライブラリ（STL、pthreadのみ）を使用すること
